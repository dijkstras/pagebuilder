Architecture Approaches

How should we build and persist the Page Builder?

A

Client-Side Only

React editor + localStorage/IndexedDB

Pages stored locally in the browser. Export/import as JSON for sharing with the team.

Pros

  • Simplest to build
  • No backend needed
  • Fast, offline-capable

Cons

  • Limited to one device
  • No real team collaboration
B

Client + Backend (Recommended)

React editor + Node/Express backend + database

Pages stored on a server. Team members can save, load, and share layouts easily.

Pros

  • Real persistence
  • Easy team sharing
  • Room to scale

Cons

  • Need backend infrastructure
  • More complex setup
C

Hybrid (Start Simple)

React editor + JSON files on disk

Pages as JSON files in a project folder. Save/load through the UI, stored locally or via Dropbox.

Pros

  • Simpler than full backend
  • Version control friendly
  • No database setup

Cons

  • Need file system access
  • Limited real-time sharing
